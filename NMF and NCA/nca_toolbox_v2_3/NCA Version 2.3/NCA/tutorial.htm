Introduction

The network component analysis (NCA) toolbox is organized as a workflow.  This means that specific steps have to occur in order for the program to run correctly.  The graphical user-interface (GUI) portion of toolbox is designed to run one NCA analysis at a time, and allow you to visualize the results. This tutorial is designed to get you to introduce you to NCA and to use the NCA Toolbox.  Most importantly, the GUI toolbox only works with MATLAB 7.0, if you require a version of the GUI to work with previous editions of MATLAB please contact us and we will make one available.  However, you should be able to run the non-GUI command line NCA programs from MATLAB prompt with previous versions of MATLAB.

I. Installing and Running the NCA Toolbox Version 1.0:
Instructions for Windows XP:
  • Unzip the NCA Toolbox Release ZIP File and extract all files to a directory.
  • From Windows Explorer, double-click on the file: NCA_Toolbox_for_Windows.exe in the "nca" subdirectory.
  • This file loads MATLAB, sets the NCA Toolbox path
  • and automatically executes the MATLAB script:  RunNCAToolbox
If for some reason, this does not work then follow the Unix/Linux instructions.

Instructions for UNIX Systems including Linux and Mac OS X:

  • Unzip the NCA Toolbox Release ZIP File and extract all files to a directory.
  • Load the MATLAB program.
  • From the command prompt within the MATLAB interpreter change to the 'nca' subdirectory within the installed NCA toolbox directory. 


For example:

>> cd nca_release_1.0/nca
Execute the NCA Toolbox GUI with the command (ignore the >>)

>> RunNCAToolbox

Note that the NCA Toolbox will automatically modify your MATLAB path to ensure the NCA program files load correctly. 
To reset your path load pathtool from the command prompt and click "default" and then "save path". 



Demo NCA Session

To get started:

  • Step 1:  Select E. coli as the organism to analyze in the pop-up box.
  • Step 2:  Next click "Load Connectivity Matrix"
  • Select the following file by clicking on it:
    • E_coli_connectivity_Kao_et_al_PNAS.mat
  • Now click "Load from Selected File".     The file should load successfully and now close the load screen and you will be returned to the main screen.
  • Step 4:  Next click "Load Microarray Data".  
    • Select the following file by clicking on it:   E_coli_Glucose_to_Acetate_Shift_Kao_et_al_PNAS.mat
  • Now click:  "Load from Selected File".  The file should load successfully and you will be returned to the main screen.
  • Step 5. Now click "Match Microarray Data and Connectivity".
    • This will match the connectivity matrix to the microarray data matrix row by row by each identifier.  A common error is that the selected organism (in Step 1) is wrong, so the matching fails.  Make sure to double check that you selected the correct. 
  • Step 5:  Now click "Select Regulatory Network". 
    • To choose a regulatory network:  Select "16 TFs" and click auto-fill. Then Click "Find NCA Network" 
    • Now Click "Save Selected Subnetwork ".  This will save the selected subnetwork to the global workspace.
  • Final Step:  Now Click the Button "Run NCA".

Now you will see a progress bar indicating the total number of initial guess iterations being run.  After NCA completes running you can explore the TFA profiles by graphical
analysis via the plot menu.  You may also cluster the TFAs using the Analyze TFA menu.  At any time you can save your workspace or load a previous workspace from the file menu. 

 
Spectra Analysis




 
First we assume that you have successfully run the demo session of NCA.  If you have not, please go to the demo tutorial of this document.  Spectra analysis assumes that you have provided pre-matched experimental and connectivity data.  This means that the matrices are lined up row-by-row and there is exactly the same number of rows in both the connectivity and expression matrices.




 
To get started, select as your organism type:  "Spectra".  Then click "Match Spectra Data to Connectivity Data", followed by "Select Regulatory Network".    Here you can choose the (regulatory) factors you want to analyze.  Save your selection to the workspace and then run NCA.   Now, the program will ask you for your choice of normalization.  Here you should enter the absorption vector (space delimited) one value for each factor, and the index of the wavelength (P) vector to normalize by.  This should be an integer greater than 0 and less than the total number of experiments.




 
For example, the index in Figure 1 is set to 1 1 1 which is the first indexed value of the decomposed spectrum.  The matching absorptions for this index are 0.3968 0.2736 0.3365. 
 
Now click OK to use these normalization settings, or click CANCEL to use the default  microarray normalization proceedure.  Generally you want to click OK if you are analyzing the hemoglobin spectra from the PNAS paper.

Connectivity and Expression Data Examples:

We have provided pre-initialized data for the NCA Toolbox.  This data consists
of a regulatory network data or expression data (signal data), and is available for
Spectra, Escherichia coli K12, Saccharomyces cerevisiae and Homo sapiens. 
These can be loaded directly from the Toolbox and are visible in the list box of the
load connectivity or load microarray screens.

 Input Data Formats:

The NCA Toolbox reads tab-delimited text files that are organized
in a specific row-column format.  That is, you should include a column header and row header for each file.  Example data files for both connectivity and expression
are provided in the ExampleData directory.
 
You cannot have missing values in either the microarray or connectivity file and the Toolbox should warn you if you do.  To fix this, you should remove any genes (rows) that have missing data before running NCA.  Alternatively, you can impute the missing values using standard preprocessing tools such as GEPAS: http://gepas.bioinfo.cnio.es/tools.html ,
save this data to a new file.  After running the GEAPS tools you first, have to load the file into  MS Excel and remove any comment lines beginning with a hash/pound sign ( symbol # ).  Save this new file in tab delimited format, and then load it into the NCA Toolbox.  Again, example files are provided in the ExampleData.

Note:
GEPAS is a web-based program to filter or impute missing values. 
We recommend k nearest neighbor (kNN) imputation and to set k=15.

Setting TF Constraints:

NCA allows you to specify zero constraints on the TFA profiles, which
is necessary, for example, when analyzing expression profiling of a regulator knockout.

To select TF constraints, after loading in the data and connectivity matrices, matching them, and selecting the regulatory network you want to analyze, then click Set TF constraints.

Step-by-step:

 

Click on Set TF constraints.

  • Add each TF and experiment pair to the list of available constraints.
  • Click Save Constraints to save the regulator knockout constraints to the global workspace.
  • Now run NCA.

How to Analyze an Organism not in the list:

With NCA you can analyze other organisms by providing prematched
connectivity and expression data. Select your organism of interest,
then click the check box “Check if Connectivity and Microarray data are prematched by Row”. Then click Match Data Matrices which will copy over the prematched data for NCA analysis.  You can select and analyze different regulatory networks using the "Select Regulators" option.


Inverting TFA Profiles to Match Known Regulations

 

As a scientist you may have prior knowledge of the sign of a TFA.  For instance you may expect the regulatory activity to increase positively and then decrease eventually hitting a steady state of zero activity.  However, the NCA solution may in fact be the exact opposite.  This is not cause for concern.  These solutions are in fact identical, except that the negative has been placed on the TFA profile rather then the CS value during optimization.  This happens mathematically, because the gene expression profile is fit as the product of CS and TFA profile.  To fix this we provide a menu screen to change each individual TFA profile.  This allows you to inspect each TFA profile and modify them accordingly.  Remember that inverting a TFA profile (or flipping) does not change the solution.

 

A Note on Microarray and Connectivity Data Format

NCA uses ORF label identifiers for E. coli and S. cerevisiae to match expression data with connectivity data. You can sometimes use the common gene name to match identifiers. For H. sapiens we use the Ensembl identifier, the NCBI Genbank identifier, or the common gene name. Microarray and expression data are matched together (that is lined up row-by-row) using these labels to find the common index. To explore the labels we have in our Genebank database, each genebank can be explored by loading it into the MATLAB spreadsheet from the subdirectory: nca/src/genebanks. However, if you are going to do this, please make a backup of the software.

 

Fast NCA

 

The Fast NCA option allows the solution to converge to a more relaxed solution by increasing the condition number approximation of the second NCA criteria as determined during optimization.  Fast NCA can be used to quickly assess the convergence properties of an analysis, but for final use, regularized NCA or normal NCA should be used.

Description of MATLAB Variables in the Global Workspace

Variable Name

Description of Variable

Ap

Final NCA computer values of CS over the selected regulators

P

Final NCA computed TFA profiles

A_rank_final

The structured matrix that satisfies NCA criteria and can be used to run NCA

E

Original Expression Data

Elabels

The row labels for matrix E

A

The original connectivity data

Alabels

The row labels for matrix A

Plabels

The column labels for matrix A

Experiment_labels

The column labels for matrix E

tfa_ids

The subset of Plabels that has connectivity in A for the genes found in Elabels

gene_ids

The subset of Elabels that matches the connectivity labels in matrix A

method

The selected NCA method:  gNCA or NCA

organism

The selected organism to analyze

iterations

The default number of iterations to run NCA before averaging and return a solution.

E0

The subset of E that matches the connectivity data

A0

The subset of A that matches the expression (or signal) data

gene_ids_final

The set of gene labels that correspond to the rows in A_rank_final

tfa_ids_final

The index vector of tfa_ids that corresponds to the columns of A_rank_final

Ep

The subset of expression data (E0) used to compute the final matrices Ap and P.

P_dev

Matrix of TFA deviations
